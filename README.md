# ImageDiff

PWA для поиска отличий на классических головоломках «найди отличия». На вход — **одно изображение**, на котором две почти одинаковые картинки (рядом или одна над другой). Приложение само находит обе картинки, сравнивает их и **подсвечивает рамками все отличия** — на обеих картинках сразу.

Работает полностью **в браузере, без сервера**: картинка никуда не загружается, вся обработка идёт на вашем устройстве. Можно «установить» на домашний экран айфона или в Dock на Mac и пользоваться офлайн.

## Возможности

- Автоматически определяет, как расположены картинки — **горизонтально** (рядом) или **вертикально** (одна над другой).
- Находит обе картинки **внутри кадра**, даже если рядом есть лишнее: баннеры, надписи («Найди 10 отличий»), декоративный фон и даже мини-ответ с уже обведёнными отличиями — лишнее игнорируется.
- Подсвечивает каждое отличие **рамкой на обеих картинках**.
- **Слайдер чувствительности** — если нашлось слишком мало или слишком много.
- **Переключатель ориентации** (авто / горизонтально / вертикально) — на случай, если авто ошиблось.
- **Сохранение** результата в PNG.
- Загрузка картинки тремя способами: выбор файла, перетаскивание (drag-and-drop), вставка из буфера (Cmd/Ctrl+V).
- **Офлайн** через service worker.

## Как запустить

Приложению нужен любой статический веб-сервер (открыть `index.html` как `file://` нельзя — браузер запрещает ES-модули и service worker по такому протоколу).

### Локально

Из корня проекта:

```bash
python3 -m http.server 8000
```

Затем откройте в браузере:

```
http://localhost:8000/app/index.html
```

Подойдёт любой аналог: `npx serve`, расширение «Live Server» в VS Code и т.п. Важно лишь, чтобы корнем раздачи был **корень проекта** (приложение в `app/` подключает ядро из `src/core/` по относительному пути `../src/core/...`).

### «Установить» как приложение

- **iPhone (Safari):** откройте страницу → «Поделиться» → «На экран „Домой"».
- **Mac (Safari):** меню «Файл» → «Добавить в Dock».
- **Chrome:** значок установки в адресной строке.

После установки работает офлайн.

## Как пользоваться

1. Откройте приложение.
2. Загрузите картинку (перетащите, выберите файлом или вставьте из буфера).
3. Приложение само всё обработает и обведёт отличия. Сверху — счётчик «Найдено отличий: N».
4. Если результат не идеален:
   - подвигайте **«Чувствительность»**: вправо — находит более мелкие/слабые отличия (но возможен шум), влево — только явные;
   - если перепутана ориентация — задайте её вручную в **«Ориентация»**.
5. Нажмите **«Сохранить»**, чтобы скачать картинку с рамками, или **«Другая картинка»**, чтобы начать заново.

**Сообщения о результате:**

| Сообщение | Что значит / что делать |
|---|---|
| `Найдено отличий: N` | Всё хорошо. |
| `Отличий не найдено…` | Повысьте чувствительность. |
| `Слишком много отличий…` | Вероятно шум — понизьте чувствительность. |
| `Не удалось распознать две картинки…` | Авто не нашло две панели (помогло бы ручное выделение — появится позже). |

## Как это работает (алгоритм)

Ключевая идея: **не резать кадр пополам**, а понять, на какой вектор сдвинута одна картинка относительно другой.

1. **Поиск смещения.** Две картинки — это одна и та же сцена, сдвинутая на постоянный вектор (вправо или вниз). Приложение сравнивает изображение с его же сдвинутой копией при разных сдвигах и берёт самый «совпадающий» вариант. Этот сдвиг сразу даёт и **ориентацию**, и **расстояние** между картинками. Лишний контент (баннеры, надписи, мини-ответ) не повторяется и не мешает.
2. **Зона сравнения.** Берётся самая большая область, где картинка и её сдвинутая копия хорошо совпадают, — это и есть две панели. Всё, что не совпало (надписи, баннеры), отбрасывается.
3. **Поиск отличий.** Внутри зоны сравнения отличия — это небольшие участки, где яркость заметно расходится. Их выделяют по порогу, чистят от шума и собирают в прямоугольники.
4. **Отрисовка.** Каждое отличие обводится рамкой на обеих панелях.

Подробности — в дизайн-доке: [`docs/superpowers/specs/2026-06-29-image-diff-design.md`](docs/superpowers/specs/2026-06-29-image-diff-design.md).

## Архитектура и структура файлов

Два слоя. **Ядро** (`src/core/`) — чистая логика без браузера: его можно один-в-один перенести в нативное приложение iOS/Mac. **Адаптер** (`app/`) — ввод-вывод и интерфейс на Canvas/DOM.

```
src/core/
  image.js        — полутон, уменьшение, размытие
  align.js        — поиск вектора смещения и ориентации
  morphology.js   — чистка бинарных масок (эрозия/дилатация)
  components.js   — связные области → прямоугольники, слияние
  diffEngine.js   — сборка конвейера: findDifferences()
app/
  index.html, app.js, style.css, manifest.webmanifest, sw.js, icon.svg
tests/            — тесты ядра на синтетических головоломках
docs/superpowers/ — спека и план реализации
```

## Программный интерфейс ядра

Ядро не зависит от браузера и работает с «сырыми» пикселями RGBA (формат как у `ImageData`).

```js
import { findDifferences } from './src/core/diffEngine.js';

// image: { data: Uint8ClampedArray (RGBA), width: number, height: number }
const result = findDifferences(image, { threshold: 45, orientation: 'auto' });
```

**Результат:**

```js
{
  status: 'ok',                         // 'ok' | 'no_diffs' | 'too_many_diffs' | 'panels_not_found'
  orientation: 'horizontal',            // итоговая ориентация ('horizontal' | 'vertical' | null)
  offset: { dx: 130, dy: 0 },           // вектор от картинки A к картинке B
  panelA: { x, y, w, h },               // прямоугольник первой картинки (или null)
  panelB: { x, y, w, h },               // прямоугольник второй картинки (или null)
  diffs: [ { x, y, w, h }, ... ]        // прямоугольники отличий в координатах картинки A
}
```

Чтобы нарисовать отличие на второй картинке, сдвиньте его рамку на `offset`: `{ x: d.x + offset.dx, y: d.y + offset.dy, w: d.w, h: d.h }`.

**Пример в браузере** (как в `app/app.js`): берём `ImageData` с canvas, прогоняем через ядро, рисуем рамки.

```js
const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);
const { diffs, offset } = findDifferences(imageData, { threshold: 44 });
for (const b of diffs) {
  ctx.strokeRect(b.x, b.y, b.w, b.h);                          // на картинке A
  ctx.strokeRect(b.x + offset.dx, b.y + offset.dy, b.w, b.h);  // на картинке B
}
```

## Все параметры (`DEFAULT_OPTIONS`)

Передаются вторым аргументом `findDifferences(image, options)`. Любой можно не указывать — возьмётся значение по умолчанию.

| Параметр | По умолч. | Диапазон | Что делает |
|---|---|---|---|
| `orientation` | `'auto'` | `'auto'` \| `'horizontal'` \| `'vertical'` | Как расположены картинки. `auto` — определить самому. Задавайте явно, если авто ошибается. |
| `threshold` | `50` | 0–255 | **Главный рычаг чувствительности.** Насколько сильно должна расходиться яркость, чтобы считать это отличием. Меньше — чувствительнее (больше находок и шума), больше — только явные отличия. В UI слайдер «Чувствительность» меняет именно его. |
| `matchThreshold` | `22` | 0–255 | Насколько *похожими* должны быть пиксели, чтобы попасть в «зону сравнения» (две панели). Влияет на поиск самих картинок, а не отличий. Обычно трогать не нужно. |
| `matchRadius` | `2` | 0–4 | Допуск ±N px при сравнении: для каждого пикселя берётся минимум разницы по окрестности в другой панели. Гасит «дрожание» перерисованных/пересжатых контуров. Больше — устойчивее к рассинхрону, но мелкие отличия могут потеряться. |
| `blurRadius` | `1` | 0–5 | Радиус предварительного размытия. Гасит «лесенку» сглаживания и мелкий шум перед сравнением. Больше — устойчивее к шуму, но теряются совсем мелкие отличия. |
| `maxWorkingDim` | `1100` | 400–4000 | Рабочее разрешение стадии различий: картинка уменьшается так, чтобы бóльшая сторона не превышала это значение (ради скорости). Меньше — быстрее, но грубее рамки. |
| `minBlobFrac` | `0.0005` | 0–0.01 | Минимальная площадь отличия как доля площади зоны сравнения. Отсеивает пылинки. Больше — игнорирует более крупные пятна. |
| `mergeGapFrac` | `0.012` | 0–0.1 | Близкие рамки сливаются в одну, если расстояние меньше этой доли от стороны зоны. Больше — агрессивнее объединяет. |
| `maxDiffs` | `60` | ≥1 | Если отличий нашлось больше — результат помечается как `too_many_diffs` (вероятно шум). |
| `alignMaxDim` | `160` | 64–512 | До какого размера уменьшать картинку при ГРУБОМ поиске смещения (ради скорости). Меньше — быстрее, но грубее. |
| `alignRefineDim` | `700` | 300–2000 | До какого размера уменьшать при УТОЧНЕНИИ смещения. Уточнение идёт не на полном разрешении (это в разы дороже, а такая точность не нужна — дальше работает `matchRadius`). |
| `alignStep` | `2` | ≥1 | Шаг подвыборки пикселей при оценке совпадения. Больше — быстрее, но менее точно. |
| `refineStep` | `2` | ≥1 | Шаг подвыборки на этапе уточнения. |
| `minSeparationFrac` | `0.15` | 0–0.5 | Минимальное расстояние между картинками (доля стороны). Не даёт принять «почти нулевой сдвиг» за ответ. |
| `minOverlapFrac` | `0.2` | 0–0.9 | Минимальное перекрытие при сравнении со сдвигом. |
| `maxSkewFrac` | `0.04` | 0–0.2 | Допустимый перекос по второй оси (для чуть неровно расположенных картинок). |
| `alignMaxScore` | `70` | 0–255 | Порог «доверия» к найденному смещению (среднее различие). Если лучшее совпадение хуже — считаем, что две картинки не нашли (`panels_not_found`). |
| `manualRegions` | `null` | `[{x,y,w,h},{x,y,w,h}]` | **Аварийный люк:** задать обе картинки вручную вместо автопоиска. Первый прямоугольник — картинка A, второй — B; смещение берётся из их разницы. (UI для этого появится в будущей версии; в ядре уже поддерживается.) |

## Тесты

Тесты написаны **осмысленно, по реальным рискам**, а не ради процента покрытия. Они работают на синтетических головоломках с заранее известными отличиями и проверяют:

- правильность определения **ориентации** и **вектора смещения** (горизонталь и вертикаль);
- что найденные рамки **накрывают** все заданные отличия и нет лишних;
- устойчивость к **лишнему контенту** (баннер не порождает ложных отличий);
- краевой случай **«нет отличий»**;
- корректность вспомогательных модулей (морфология, связные области, утилиты изображения).

Запуск (нужен только Node ≥ 18, зависимостей нет):

```bash
npm test
```

## Ограничения и планы

**Сейчас (v1):**
- Рассчитано на **чистые цифровые картинки** (скриншоты, изображения из интернета). Геометрия двух картинок должна быть одинаковой.
- Режим вывода — рамки.

**На будущее:**
- **Фото распечаток/книг** (перспектива, блики, тени, перекос) — потребуется выравнивание с поворотом.
- **Ручное выделение** двух картинок (UI для `manualRegions`).
- Дополнительные режимы вывода: заливка, показ отличий по одному.
- **Нативные приложения iOS/Mac** на том же ядре.
